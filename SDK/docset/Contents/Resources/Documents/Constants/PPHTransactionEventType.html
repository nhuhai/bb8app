Constants

ePPHTransactionType_Idle

The TransactionManager has entered the idle state. It is not doing any work and is ready to begin a payment or refund.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_GettingPaymentInfo

The transactionManager has entered a state of data collection. beginPayment() has been called and we
are currently collecting information from card readers or the application. You can now set/modify the invoice, signatures, and other data related to this payment.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_DidStartReaderDetection

This event will be triggered in cases where reader detection takes a while, such as for the audio readers. It presents an opportunity to show UI indicating that you are “working on it”

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_DidDetectReaderDevice

A fully working reader was detected and is available

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_DidRemoveReader

A reader device has been removed from the system

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_CardReadBegun

Something has occurred in the read head of the reader. Since processing can take a second or so, this allows you to get some UI up. Be careful how much work you do here because taxing the CPU will hurt success rate.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_CardDataReceived

Card data received. The transaction manager has received card data, and it can be processed

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_ReadCardNotAllowed

Card data received. The transaction manager has received card data, but it is not allowed to process it

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_FailedToReadCard

A swipe attempt failed. Usually this means the magstripe could not be read and the merchant should try again.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_ProcessingPayment

The TransactionManager has entered a state where it is communicating with the backend servers to collect a payment. It is now too late to cancel the payment.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_WaitingForSignature

The TransactionManager has entered a state where it has completed the transaction successfully and is waiting for the app to collect a signature from the customer and call the finalizePayment API.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_TransactionCancelled

Event specifying that transaction has been cancelled. This will happen only in case of EMV Payments when te user pulls the card out of the reader when transaction is going on

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_TransactionDeclined

Event specifying that transaction has been declined. This will happen only in case of EMV Payments in case terminal declines the transaction for some reason.

Declared In PPHTransactionManagerDelegate.h.

ePPHTransactionType_RefundTypeSelectionCancelled

Refund type selection has been cancelled.

Declared In PPHTransactionManagerDelegate.h.
